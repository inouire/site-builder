Site-builder is a simple example of a static site generator. These are tools that generate websites from one or more content files by wrapping them in one or more templates. Why?

  • By keeping your layout separate from your content, both are easier to maintain and update.

  • Generating static HTML files and then uploading them is much more efficient than loading your content into your template for each visitor to your site. Your web server can typically serve plain HTML well over 100 times faster than it can serve even simple PHP.

  • Static site generators are much more simple and manageable than CMSes, which makes them good projects for beginner programmers to tinker with.

  • Because you can run your generator offline on your home computer, and then upload the published HTML to your website, you automatically have an off-site backup of your content. By using static HTML instead of dynamic PHP, you also reduce the potential attack surface of your website.

What's good about site-builder

Site-builder is a very simple generator. In its basic form, you write your page content as simple HTML or Markdown files and put them in the content folder, edit the template to look how you want it to, and run php sitebuilder.phar rebuild. It uses easy and powerful Twig template language.

Site-builder is totally free and open-source. Play around with it, extend it, and learn from it.

What's not so great about site-builder

Site-builder has a very short feature list. If you want something that dynamically creates your site's navigation, writes multiple entries to one output file (like a blog), creates next/previous links, or anything like that, look at Jekyll (the Ruby-based SSG used by GitHub's site pages), Phrozn (another PHP SSG), or Hyde (Python, very fancy).

Quick start

  1. Download the .phar file. It's the whole app in one file, with everything it needs to run.
  2. Put sitebuilder.phar in the directory where you want to keep the installation.
  3. Run php sitebuilder.phar init to create directories, config, and sample files.
  4. Test your installation with php sitebuilder.phar rebuild. If it works, you should see a generated example.html file in the output directory.
  5. Replace the default template with your own. Twig is a pretty straightforward templating language; just put {{ content | raw }} where you want your page content to appear. Check out the README.md file for more help and links to the Twig docs.
  6. Create your content files. You can write plain HTML (and save as .html or .htm), or Markdown (and save as .md or .markdown). You can make sub-directories too.
    Check out the README.md file for more info on Markdown, including a link to the documentation. You can hand more information to your templates by adding a "front matter" block. Again, see the README file for more information about this.
  7. Run php sitebuilder.phar rebuild to regenerate your site.
